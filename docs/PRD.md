# 무드뮤직 — 제품 요구사항 문서 (PRD)

> **버전**: v1.0  
> **작성일**: 2026-05-11  
> **스코프**: MVP (4주)  
> **플랫폼**: 앱인토스 미니앱

---

## 1. Executive Summary

### 문제 정의

음악 스트리밍 서비스의 추천은 장르/아티스트/청취 이력에 의존한다. 사용자가 "지금 이 기분"에 맞는 음악을 찾으려면 직접 검색하거나 큐레이션 플레이리스트를 뒤져야 하며, 이 과정에서 이탈이 발생한다.

### 제안 솔루션

화면에 떠다니는 감정 키워드 2~4개를 선택하면, LLM이 조합을 분석하여 YouTube Music 기반 플레이리스트를 즉시 생성하는 앱인토스 미니앱. 앱 설치 없이 토스 내에서 3초 안에 첫 추천을 받을 수 있다.

### 성공 지표 (출시 후 4주 기준)

| KPI | 목표값 | 측정 방법 |
|-----|--------|-----------|
| DAU | 1,000+ | 앱인토스 분석 대시보드 |
| D7 리텐션 | 30% 이상 | 코호트 분석 |
| 추천 만족도 | 70% 이상 | 인앱 3단계 리액션 피드백 (😐/🙂/😍) |
| 프리미엄 전환율 | 3% 이상 | 인앱결제 전환 추적 |
| 첫 추천까지 소요 시간 | 3초 이내 | 키워드 선택 완료 → 첫 곡 재생 시작 |

## 2. User Experience & Functionality

### 2.1 유저 페르소나

| 페르소나 | 설명 | 핵심 니즈 |
|----------|------|-----------|

### 2.2 유저 스토리 & 수락 기준

#### US-01: 감정 키워드 선택

> As a 사용자, I want to 화면에 떠다니는 감정 키워드 버블 중 2~4개를 탭하여 선택 so that 내 현재 기분을 표현할 수 있다.

**수락 기준:**
- [ ] 화면에 30~50개 키워드가 플로팅 버블 형태로 표시된다
- [ ] 버블은 물리 시뮬레이션으로 자연스럽게 떠다닌다
- [ ] 최소 2개, 최대 4개까지 선택 가능하다
- [ ] 선택된 버블은 시각적으로 구분된다 (확대 + 색상 변경)
- [ ] 1개 선택 시 "1개 더 골라주세요" 안내가 표시된다
- [ ] 4개 초과 선택 시도 시 가장 먼저 선택한 것이 해제된다
- [ ] 앱 진입 후 3초 이내에 키워드 선택이 가능한 상태여야 한다 (로딩 포함)

#### US-02: AI 기반 플레이리스트 생성

> As a 사용자, I want to 선택한 키워드 조합에 맞는 플레이리스트를 자동으로 받아 so that 직접 검색하지 않아도 기분에 맞는 음악을 들을 수 있다.

**수락 기준:**
- [ ] 키워드 2~4개 선택 완료 후 "추천 받기" 버튼을 탭하면 플레이리스트가 생성된다
- [ ] 플레이리스트는 5~10곡으로 구성된다
- [ ] 추천 생성 시간은 2초 이내 (캐시 히트 시 500ms 이내)
- [ ] 플레이리스트 상단에 `🤖 AI 추천` 배지가 표시된다
- [ ] 동일 키워드 조합으로 재요청 시 캐시된 결과를 반환한다
- [ ] LLM이 키워드를 YouTube 최적화 검색 쿼리로 변환한다 (예: "잔잔한 + 새벽 + 혼자" → "lo-fi chill late night alone ambient")

#### US-03: YouTube 음악 재생

> As a 사용자, I want to 추천된 곡을 앱 내에서 바로 재생 so that 별도 앱 전환 없이 음악을 들을 수 있다.

**수락 기준:**
- [ ] YouTube iframe으로 곡이 재생된다
- [ ] 재생/일시정지/다음곡/이전곡 컨트롤이 제공된다
- [ ] 현재 재생 중인 곡의 제목/아티스트가 표시된다
- [ ] 플레이리스트 내 곡 목록을 확인하고 특정 곡을 선택할 수 있다
- [ ] 광고 재생 시 음악이 자동으로 일시정지된다

#### US-04: 감정 히스토리 저장

> As a 사용자, I want to 과거에 선택한 키워드 조합과 들은 곡을 다시 볼 수 so that 좋았던 조합을 다시 사용할 수 있다.

**수락 기준:**
- [ ] 토스 로그인 기반 유저 식별키로 히스토리가 저장된다
- [ ] 최근 20개 키워드 조합 히스토리가 날짜별로 표시된다
- [ ] 히스토리에서 과거 조합을 탭하면 동일 조합으로 재추천된다
- [ ] Storage API를 사용하여 데이터가 저장된다

#### US-05: 미니 리액션 피드백

> As a 사용자, I want to 추천된 플레이리스트에 간단한 피드백을 남길 수 so that 추천 품질이 개선될 수 있다.

**수락 기준:**
- [ ] 플레이리스트 재생 완료 후 3단계 리액션 (😐/🙂/😍) 선택 UI가 표시된다
- [ ] 피드백은 선택 즉시 저장된다 (별도 제출 버튼 없음)
- [ ] 피드백 데이터는 키워드 조합 + 결과 곡 목록과 함께 저장된다

#### US-06: 보상형 광고 기반 새로고침

> As a 무료 유저, I want to 보상형 광고를 시청하고 새로운 추천을 받을 수 so that 일일 제한 내에서도 더 많은 음악을 탐색할 수 있다.

**수락 기준:**
- [ ] 무료 유저는 일 5회 키워드 조합 제한이 있다
- [ ] 제한 도달 시 "광고 보고 추천 받기" 버튼이 표시된다
- [ ] 보상형 광고 시청 완료 후 즉시 1회 추가 조합이 가능하다
- [ ] 광고 로딩은 사전 로딩 방식이다 (실시간 로딩 금지)

#### US-07: 프리미엄 구독

> As a 사용자, I want to 프리미엄 구독으로 광고 없이 무제한 사용 so that 끊김 없는 경험을 할 수 있다.

**수락 기준:**
- [ ] 인앱결제로 월 1,900~2,900원 구독이 가능하다
- [ ] 프리미엄 유저는 광고가 완전히 제거된다
- [ ] 프리미엄 유저는 키워드 조합 횟수 제한이 없다
- [ ] 프리미엄 유저는 감정 캘린더 풀 히스토리에 접근 가능하다
- [ ] 프리미엄 유저는 프리미엄 키워드 팩 (계절/상황별)을 사용할 수 있다

#### US-08: 생성형 AI 사전 고지

> As a 서비스 운영자, I want to 생성형 AI 사용 사실을 사전 고지 so that 앱인토스 정책을 준수하고 과태료 리스크를 제거할 수 있다.

**수락 기준:**
- [ ] 최초 이용 시 바텀시트로 "AI를 활용하여 키워드를 분석하고 음악을 추천합니다" 고지
- [ ] 바텀시트에 "확인" 버튼이 있으며, 확인 전까지 서비스 이용 불가
- [ ] 고지 확인 여부는 Storage API에 저장되어 재방문 시 미노출

### 2.3 Non-Goals (MVP에서 제외)

- 소셜 무드 매칭 (같은 기분 유저 연결)
- B2B 공간 무드 큐레이션
- 실시간 협업 필터링 (WebSocket)
- 엣지 추론 경량 모델 (ONNX)
- 키워드-오디오 임베딩 매핑 (CLAP)
- 감정 캘린더 UI (히스토리 목록만 제공, 캘린더 뷰는 v2)
- 흔들어서 리셋 기능
- 시간대/날씨 자동 키워드 제안 (Week 4에 기본 구현만 포함)

## 3. AI 시스템 요구사항

### 3.1 LLM 키워드 확장 엔진

**역할**: 사용자가 선택한 감정 키워드 2~4개를 YouTube 검색에 최적화된 영문 쿼리로 변환

**입력**: `["잔잔한", "새벽", "혼자"]`  
**출력**: `"lo-fi chill late night alone ambient korean"`

#### 요구사항

| 항목 | 기준 |
|------|------|
| 응답 시간 | 캐시 미스 시 1.5초 이내, 캐시 히트 시 50ms 이내 |
| 캐싱 전략 | 키워드 조합(정렬된 Set)을 키로 결과 캐싱. TTL 24시간 |
| 출력 형식 | YouTube 검색 최적화 영문 쿼리 (1~3개 쿼리 변형 생성) |
| 필터링 | 부적절한 콘텐츠 키워드 제외 (욕설, 폭력, 성인) |
| 폴백 | LLM 응답 실패 시 키워드 직접 번역 + 장르 매핑 테이블 사용 |

#### 평가 전략

| 메트릭 | 목표 | 측정 방법 |
|--------|------|-----------|
| 검색 결과 적합도 | 70% 이상 | 50개 키워드 조합 → YouTube 검색 → 상위 5곡 중 감정 매칭 비율 |
| 쿼리 다양성 | 3개 변형 중 2개 이상 상이 | 동일 조합 반복 요청 시 결과 중복률 30% 미만 |
| 유해 콘텐츠 필터링 | 100% | 금칙어 포함 쿼리 생성 건수 0건 |

### 3.2 생성형 AI 정책 준수

| 의무 | 구현 방법 | 위반 시 |
|------|-----------|---------|
| 사전 고지 | 최초 이용 시 바텀시트 | 최대 3,000만 원 과태료 |
| 결과물 표시 | 플레이리스트 상단 `🤖 AI 추천` 배지 | 최대 3,000만 원 과태료 |

## 4. Technical Specifications

### 4.1 아키텍처 개요

```
[사용자] → [플로팅 버블 UI] → [키워드 2~4개 선택]
                                        ↓
                              [LLM 키워드 확장 (서버)]
                                        ↓
                              [YouTube 검색 쿼리 생성]
                                        ↓
                              [YouTube Data API v3]
                                        ↓
                              [결과 필터링 + 정렬]
                                        ↓
                              [YouTube iframe 재생]
```

### 4.2 기술 스택

| 레이어 | 선택 | 비고 |
|--------|------|------|
| 프레임워크 | Vite 기반 Web (WebView) | 앱인토스 기본 |
| UI | TDS (Toss Design System) + 커스텀 버블 UI | 물리 시뮬레이션 라이브러리 필요 |
| SDK | `@apps-in-toss/web-framework` | 광고/결제/로그인 통합 |
| 인증 | 토스 로그인 → 유저 식별키 | 플랫폼 유일 옵션 |
| 저장 | Storage API | 감정 히스토리, 좋아요, 설정 |
| 재생 | YouTube iframe API | 앱인토스 iframe 예외 허용 |
| 검색 | YouTube Data API v3 | 일 10,000 units quota |
| AI | LLM 서버사이드 (키워드 확장) | 결과 캐싱 필수 |
| 광고 | GoogleAdMob via 앱인토스 SDK | 전면형/보상형/배너 |
| 결제 | 앱인토스 인앱결제 SDK | 디지털 상품 (프리미엄 구독) |

### 4.3 Integration Points

#### YouTube Data API v3

| 항목 | 상세 |
|------|------|
| 엔드포인트 | `search.list` (type=video, videoCategoryId=10) |
| Quota | 일 10,000 units (search.list = 100 units/call) |
| 대응 | 인기 키워드 조합 결과 캐싱. 동일 조합 캐시 히트 |
| 필터 | `videoCategoryId=10` (Music) + `topicId` 활용 |
| 제한 | 백그라운드 재생 불가 (YouTube 정책) |

#### 앱인토스 SDK

| 기능 | API |
|------|-----|
| 로그인 | 토스 로그인 → 유저 식별키 |
| 저장 | Storage API (key-value) |
| 광고 (보상형) | `GoogleAdMob.loadAppsInTossAdMob()` |
| 광고 (전면형) | `GoogleAdMob.showAppsInTossAdMob()` |
| 결제 | 인앱결제 SDK |
| 푸시 | 스마트 발송 (날씨/시간 세그먼트) |

### 4.4 Security & Privacy

| 항목 | 처리 방법 |
|------|-----------|
| 개인정보 | 유저 식별키만 사용. 이름/전화번호 등 미수집 |
| 데이터 저장 | 키워드 히스토리, 피드백만 Storage API에 저장 |
| AI 입력 | 감정 키워드만 LLM에 전달. 개인정보 미포함 |
| API 키 보호 | YouTube API 키는 서버사이드에서만 사용 |
| 생성형 AI 고지 | 앱인토스 정책 준수 (사전 고지 + 결과물 표시) |

## 5. Risks & Roadmap

### 5.1 MVP 로드맵 (4주)

| 주차 | 마일스톤 | 핵심 산출물 |
|------|----------|-------------|
| **Week 1** | 기획 & 기반 | 앱인토스 앱 등록, 버블 UI 프로토타입, 키워드 목록 30~50개, 프로젝트 셋업 |
| **Week 2** | 핵심 기능 | YouTube API 연동, LLM 키워드 확장, iframe 플레이어, 검색 품질 테스트 |
| **Week 3** | 연동 & 수익화 | 토스 로그인, Storage API, 광고 3종 연동, 인앱결제, 히스토리 |
| **Week 4** | 출시 준비 | 날씨/시간 API 기본 연동, AI 고지 UI, 체크리스트 확인, `.ait` 빌드 → 검수 요청 |

### 5.2 기술 리스크

| 리스크 | 영향도 | 발생 확률 | 대응 전략 |
|--------|--------|-----------|-----------|
| YouTube API Quota 초과 (일 10,000 units) | 높음 | 중간 | 인기 조합 캐싱 + quota 모니터링 알림. DAU 1,000 기준 유저당 일 10회 = 10,000 → 캐시 히트율 80% 목표 |
| LLM 응답 지연/실패 | 중간 | 낮음 | 폴백: 키워드 직접 번역 + 장르 매핑 테이블. 타임아웃 2초 |
| 모바일 물리 시뮬레이션 성능 | 중간 | 중간 | 저사양 기기 감지 → 버블 개수 축소 (50→20개). requestAnimationFrame 최적화 |
| YouTube 검색 결과 품질 불일치 | 높음 | 중간 | 검색 쿼리 3개 변형 생성 → 결과 병합 + 중복 제거. `videoCategoryId=10` 필터 필수 |
| 앱인토스 검수 반려 | 높음 | 낮음 | 출시 체크리스트 사전 전수 확인. AI 고지/광고 정책 100% 준수 |
| 백그라운드 재생 불가 (YouTube 정책) | 낮음 | 확정 | MVP에서는 포그라운드 재생만 지원. 향후 YouTube Music Premium 유도 검토 |

### 5.3 의존성
